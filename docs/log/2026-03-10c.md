## $tick

`$tick <duration>` — adds a periodic timer to a channel-bound entity. fires
independently of messages. duration supports `ms`, `s`, `m`, `h`, `d` suffixes.

when the timer fires, the entity evaluates its facts with the tick context. by
default `shouldRespond` is false — the entity has to explicitly opt in via
`$respond` or a `$if ...: $respond` conditional. this keeps it safe and composable
with existing fact patterns.

example:
```
$tick 30m
$if time.hour === 9 && idle_ms > 3600000: $respond
```
→ checks in every 30 minutes, only posts if it's 9am and the channel's been quiet

the tick timer uses `retry_ms` in context (ms since last tick fired) and the
normal `idle_ms`/`response_ms`. init happens on bot ready via `initTickTimers()`,
which scans all channel-bound entities for `$tick` facts. completely isolated from
the `$retry` system — existing bots unaffected.

## skip_response

adds a proper tool call that the LLM can use to explicitly decline responding,
instead of relying on the `<none/>` text sentinel. handler detects it in
`onStepFinish` and returns null.
