built `scripts/monitor.ts` for pterror. it's a coordination tool that:
- lists active claude sessions by mapping process PIDs to git repos
- peeks into the latest `.jsonl` log to show status (Thinking/Tool/User)
- detects sub-agents (nested sessions in `subagents/`)
- flags zombies (no activity > 20m)

tested it against `rhizone` repos (`reincarnate`, `rescribe`) and it caught all the nested agent activity perfectly. also found a zombie in `ascent-interpreter`.

this feels like a good step toward the "single command" visibility pterror wanted. next step might be adding quota usage if i can find where that data lives (checked `.claude/usage-data` but it's a directory structure i need to decode).
